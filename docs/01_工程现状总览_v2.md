## 2. 当前到底在做什么
围绕 AI 宠物项目，当前工程做了两层：

1. 官方能力层（不重造轮子）
- `services/xiaozhi-esp32-server`：核心会话服务 + 管理后台 + 管理 API
- `services/xiaozhi-mqtt-gateway`：设备 MQTT/UDP 与 WebSocket 的桥接
- `services/mcp-endpoint-server`：官方 MCP 接入点（路径 C）
- `services/voiceprint-api`：官方声纹服务
- `firmware/xiaozhi-esp32`：设备固件

2. 本地增强层（你自己的效率工具）
- `scripts/`：启动、停止、同步 IP、诊断、验收、打包
- `apps/visual-command-ui/`：可视化控制台（执行脚本白名单）
- `tools/mac/`：双击 `.command` 入口
- `.cache/dist/`：DMG 产物

补充说明：
- 当前仓库的有效运行目录只在 `services/`、`apps/`、`firmware/` 下。
- 根目录若出现 `xiaozhi-esp32-server/`、`mcp-endpoint-server/`、`voiceprint-api/`、`ai_pet_provisioner/`，都视为历史迁移残留，不参与当前部署。
