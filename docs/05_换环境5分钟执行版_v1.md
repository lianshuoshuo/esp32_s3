# 换环境 5 分钟执行版（v1）

更新时间：2026-03-19

> 这份只保留最短动作。  
> 适合两种场景：`本机 IP 变化`、`准备切到云服务器`。

---

## 1. 本机 IP 变化：直接照做

### 第一步：查新 IP
```bash
ipconfig getifaddr en0 || ipconfig getifaddr en1
```

### 第二步：同步服务
```bash
cd /Users/lss/Desktop/AI_MCP
./scripts/同步本机IP.sh
./scripts/重启本地服务.sh
./scripts/查看服务状态.sh
```

### 第三步：看这 4 个参数
- `server.fronted_url`
- `server.ota`
- `server.websocket`
- `server.mqtt_gateway`

### 第四步：如果设备还显示旧地址
直接重刷：
```bash
cd /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32
bash /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/命令脚本/清空并重刷.command --yes
```

### 第五步：重新配网验证
看两件事：
1. 屏幕/播报是否还是旧域名
2. 后台绑定是否正常

---

## 2. 切云服务器：最短动作

### 第一步：先定 4 个正式地址
- 前端地址：`https://你的域名`
- OTA：`https://你的域名/xiaozhi/ota/`
- WebSocket：`wss://你的域名/xiaozhi/v1/`
- MQTT：`mqtt.你的域名:1883`

### 第二步：改服务端参数
至少改：
- `server.fronted_url`
- `server.ota`
- `server.websocket`
- `server.mqtt_gateway`
- `server.udp_gateway`
- `server.mqtt_manager_api`
- `server.mcp_endpoint`

### 第三步：改固件默认 OTA
至少改这 2 处：
- `/Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/sdkconfig`
- `/Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/main/Kconfig.projbuild`

需要看到：
```text
CONFIG_OTA_URL="https://你的域名/xiaozhi/ota/"
```

### 第四步：重新编译并检查构建产物
```bash
cd /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32
bash /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/命令脚本/一键编译.command
rg -n "CONFIG_OTA_URL" \
  /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/build_esp32s3_v552/config/sdkconfig.h \
  /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/build_esp32s3_v552/config/sdkconfig.cmake
```

### 第五步：清空并重刷
```bash
bash /Users/lss/Desktop/AI_MCP/firmware/xiaozhi-esp32/命令脚本/清空并重刷.command --yes
```

### 第六步：重新配网
验证：
1. 设备显示的是你的域名，不是 `xiaozhi.com`
2. 后台绑定成功

---

## 3. 只记这 3 句话

1. **先改服务端参数**
2. **再改固件实际编译进去的 `CONFIG_OTA_URL`**
3. **最后清空并重刷，否则旧缓存会继续干扰**

---

## 4. 如果又看到 `xiaozhi.com`

不要乱猜，按这个顺序查：
1. `server.fronted_url`
2. 构建产物里的 `CONFIG_OTA_URL`
3. 设备有没有清空整片 Flash
