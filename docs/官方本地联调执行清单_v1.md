## 6. MCP 路径 C 验收（MCP 接入点）

### 6.1 先确认部署模式

你当前工程是**全模块部署**（manager-api + manager-web + xiaozhi-server），
推荐按官方全模块方式配置：

- 参数字典写 `server.mcp_endpoint = http://你的IP:8004/mcp_endpoint/health?key=...`
- 智能体实际下发给设备的是 `ws://.../mcp/?token=...`（由 manager-api 自动计算）

仅在**单模块部署**时，才直接在 `data/.config.yaml` 写 `mcp_endpoint: ws://.../mcp/?token=...`。

### 6.2 全模块配置与验收

1. 启动 `mcp-endpoint-server`（默认端口 `8004`）。
2. 从日志拿到 `health?key=...` 地址，替换成你的局域网 IP。
3. 写入参数 `server.mcp_endpoint`。
4. 执行联调：

```bash
cd /Users/lss/Desktop/AI_MCP
./scripts/重启本地服务.sh
LOG_LINES=200 ./scripts/查看服务日志.sh server
```

额外检查（强烈建议）：

```bash
cd /Users/lss/Desktop/AI_MCP
./scripts/一键联调验收.sh
```

在全模块模式下，看到下面两点即可判定 C 已落地：
- `server.mcp_endpoint（全模块）已配置`
- `MCP接入点健康接口 可访问（HTTP 200）`

## 8. 常见故障与最快修复

### 8.1 设备能说话但后台偶发离线
先执行：

```bash
cd /Users/lss/Desktop/AI_MCP
./scripts/同步本机IP.sh
./scripts/重启本地服务.sh
```

原因通常是局域网 IP 变化后，`sys_params` 与网关配置未同步。

### 8.2 `mcp接入点不符合规范`
单模块场景下，基本都是 `mcp_endpoint` 填成了：
- `health?key=...` 地址（错）
- `/call/` 地址（错）

应改回 `/mcp/` 的 ws 地址。

全模块场景下，请检查 `server.mcp_endpoint` 是否是 `health?key=...`，并且不是 `localhost`。

### 8.3 MCP 工具没有出现在函数列表
按顺序检查：
1. 对应路径是否初始化成功（看日志关键字）  
2. MCP 服务是否返回 `tools/list`  
3. 工具名和参数 schema 是否合规  
4. 重启后是否仍缺失（排除缓存/临时连接问题）
