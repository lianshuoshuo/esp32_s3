## 6. 常见问题与处理
### 问题 A：设备显示离线，但实际可对话
优先处理：
1. 执行 `./scripts/同步本机IP.sh`
2. 再执行 `./scripts/查看服务状态.sh`，确认参数一致
3. 如果仍异常，执行 `./scripts/一键联调验收.sh` 看失败项

### 问题 B：`verification_code_not_found` 或绑定异常
排查方向：
1. 确认设备端使用的是本地服务地址，不是历史云端地址
2. 确认 `server.fronted_url` 与当前 IP 一致
3. 重启后复测：`./scripts/重启本地服务.sh`
4. 如果设备屏幕或播报仍出现 `xiaozhi.com`，直接看：
   - `docs/03_固件切换到自有服务排障与操作手册_v1.md`

### 问题 E：固件仍显示 `xiaozhi.com`
优先不要怀疑页面，先怀疑这三层：
1. 服务端参数是否已经同步到当前 IP
2. 固件实际编译生效的 `CONFIG_OTA_URL` 是否仍是官方地址
3. 设备是否已经“清空整片 Flash + 重刷”

推荐直接按专项手册处理：
- `docs/03_固件切换到自有服务排障与操作手册_v1.md`
- 如果只是想快速执行，先看：
  - `docs/05_换环境5分钟执行版_v1.md`

### 问题 C：MCP 接入点不工作
排查方向：
1. 端口检查：`8004` 是否监听
2. 参数检查：`server.mcp_endpoint` 是否为有效地址且可访问
3. 日志检查：`./scripts/查看服务日志.sh all --follow`

### 问题 D：管理页中“部署就绪检查”显示 401
说明：
- 该检查接口受后台登录态保护，未登录时返回 401 属于正常。
