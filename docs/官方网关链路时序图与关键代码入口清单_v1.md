# 官方网关链路时序图与关键代码入口清单（v1）

## 1. 目标与范围
本清单覆盖 `xiaozhi-mqtt-gateway` 主链路：

- 设备 MQTT 登录与协议解析
- `hello` 建链：MQTT/UDP ↔ WebSocket
- 音频双向转发（UDP 加解密 + WebSocket 二进制）
- MCP 转发与工具缓存
- 管理 API（设备指令下发、在线状态查询）

---

## 2. 主链路时序图（MQTT/UDP 到 WebSocket）
```mermaid
sequenceDiagram
    autonumber
    participant Dev as 设备(固件)
    participant GW as xiaozhi-mqtt-gateway
    participant WS as xiaozhi-server(ws://.../xiaozhi/v1/?from=mqtt_gateway)

    Dev->>GW: MQTT CONNECT(clientId/username/password)
    GW->>GW: MQTTProtocol.parseConnect + validateMqttCredentials
    GW-->>Dev: CONNACK

    GW->>Dev: MCP initialize/tools/list 预取(缓存工具)
    Dev-->>GW: MCP initialize/tools/list 结果
    GW->>GW: 缓存 mcpCachedInitialize + mcpCachedTools

    Dev->>GW: MQTT PUBLISH {type:"hello",version:3,...}
    GW->>GW: parseHelloMessage() 生成 UDP key/nonce(aes-128-ctr)
    GW->>WS: WebSocketBridge.connect() + headers(device-id/client-id/auth)
    WS-->>GW: hello(session_id,audio_params)
    GW-->>Dev: MQTT hello(transport=udp,udp server/port/key/nonce)

    loop 上行音频
        Dev->>GW: UDP 音频包(16字节头+AES密文)
        GW->>GW: 解密+序列检查+修正timestamp
        GW->>WS: WebSocket 二进制音频(16字节头+opus)
    end

    loop 下行音频
        WS-->>GW: WebSocket 二进制音频(16字节头+opus)
        GW->>GW: 取timestamp/opusLength
        GW-->>Dev: UDP 音频包(AES加密后)
    end

    WS-->>GW: JSON(stt/tts/mcp/...)
    GW-->>Dev: MQTT JSON
    Dev->>GW: MQTT JSON
    GW->>WS: WebSocket JSON
```

## 3. MCP 缓存与转发时序（网关特色）
```mermaid
sequenceDiagram
    autonumber
    participant Dev as 设备(MQTT侧MCP)
    participant GW as MQTTConnection
    participant WS as WebSocketBridge(上游服务端)

    Note over GW: 连接建立后先初始化设备工具缓存
    GW->>Dev: sendMcpRequest(initialize)
    Dev-->>GW: result(initialize)
    GW->>Dev: sendMcpRequest(tools/list, cursor...)
    Dev-->>GW: result(tools/list)
    GW->>GW: 保存 mcpCachedInitialize + mcpCachedTools

    WS-->>GW: mcp notifications/initialized 或 tools/list
    GW->>GW: onMcpMessageFromBridge() 命中缓存
    GW-->>WS: 直接回填缓存结果(不再请求设备)

    WS-->>GW: mcp tools/call
    GW-->>Dev: 透传 tools/call
    Dev-->>GW: mcp result/error
    GW-->>WS: 回传 result/error
```

## 4. 管理 API 时序（本地运维调用）
```mermaid
sequenceDiagram
    autonumber
    participant Admin as 管理端脚本/UI
    participant API as Gateway Admin API(:8007)
    participant GW as MQTTConnection
    participant Dev as 设备

    Admin->>API: POST /api/commands/:clientId + Bearer(日签名)
    API->>API: authenticateRequest() 校验当日token
    API->>GW: sendMcpRequest(method, params, timeout)
    GW-->>Dev: MQTT mcp请求
    Dev-->>GW: MQTT mcp响应
    GW-->>API: result/error
    API-->>Admin: success/data 或 error

    Admin->>API: POST /api/devices/status {clientIds:[...]}
    API->>GW: getConnectionById(clientId)
    API-->>Admin: {isAlive, exists}
```

## 6. 当前实现里的关键注意点（排障优先看）
- `WebSocket` 上游地址要带 `?from=mqtt_gateway`，否则服务端不会走网关音频头处理分支。
  - 参考：`/Users/lss/Desktop/AI_MCP/services/xiaozhi-esp32-server/docs/mqtt-gateway-integration.md:10`
- `SERVER_SECRET` 必须与服务端认证密钥一致（`server.auth_key` 或 manager-api secret 体系下的等效密钥），否则 WS 鉴权失败。
  - 参考：`/Users/lss/Desktop/AI_MCP/services/xiaozhi-mqtt-gateway/app.js:64`
- 管理 API token 每天变化，按当天日期计算。
  - 参考：`/Users/lss/Desktop/AI_MCP/services/xiaozhi-mqtt-gateway/app.js:828`
- 管理 API 的状态查询参数是 `clientIds`（不是 `deviceIds`）。
  - 参考：`/Users/lss/Desktop/AI_MCP/services/xiaozhi-mqtt-gateway/app.js:912`
